### 001. 端口
* **端口号在一定程度下是约定俗成的**
| 端口   | 常见用途                |
| ---- | ------------------- |
| 80   | HTTP 正式服务（公网）       |
| 443  | HTTPS 正式服务（公网）      |
| 8080 | 本地 HTTP 测试（Java 常见） |
| 8443 | 本地 HTTPS 测试         |
| 3000 | Node/React 本地开发端口   |
| 5000 | Flask/Python 本地开发   |
| 5173 | Vite 本地开发           |
* **总结**
  - 8443 用于 HTTPS 测试服务，是一种惯例，尤其常见于 Java 系生态。
  - 3000 更偏向前端或 Node.js 本地开发使用，通常是 HTTP 而非 HTTPS。
  - 你完全可以修改端口，比如让 HTTPS 服务跑在 3000，只是这样会违背某些默认配置或安全策略。
